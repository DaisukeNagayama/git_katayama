- リモートリポジトリ
    GitHub上のデータを保存しておく場所（バックアップ的な）

- ローカルリポジトリ
    GitHubに送信するデータを登録しておく場所

    枝分かれ
    失敗してもいい用にコピーをもとに作業

    結合する

【ローカルリポジトリの作成】

0) ローカルリポジトリにしたいディレクトリ（フォルダ）を作成してそこへ移動

  $ mkdir ディレクトリ名
  $ cd ディレクトリ名

1) ローカルリポジトリの作成
  
  $ git init
  
2) ローカルリポジトリにファイルの変更点を追加（インデックスに追加）
  
  $ git add ファイル名

3) ローカルリポジトリにインデックスに追加したファイルを登録
  
  $ git commit -m "変更点などのコメント"

4) 追加したインデックス（ファイルの変更点など）を GitHub に作成

  $ git remote add origin リポジトリのURL

5) ローカルリポジトリのファイルを GitHub のリポジトリに送信

  $ git push origin master

  [設定によってはユーザー名とパスワードが要求される]

---
【ファイルの変更を GitHub（リモートリポジトリ）に保存する】

1) 変更をインデックスに追加

  $ git add ファイル名

2) ファイルを登録（コミット）

  $ git commit -m "変更点などのコメント"

3) データの送信

  $ git push origin master

---
【ブランチを作って作業する】

1) ブランチを作成する

  $ git branch ブランチ名

2) 今あるブランチを確認する

  $ git branch

3) ブランチを移動する

  $ git checkout ブランチ名

4) インデックスに変更点を登録

  $ git add ファイル名

5) ファイルをローカルリポジトリにコミット

  $ git commit -m "変更点などのコメント"

6) GitHub にローカルリポジトリのデータをプッシュ

  $ git push origin master ブランチ名

---
【ブランチを結合する】

1) 取り込み先のブランチを選択する

  $ git checkout master

2) 結合したいブランチを選択する

  $ git merge ブランチ名

3) 結合情報を GitHub にプッシュ

  $ git push origin master

---
【ブランチを削除する】

1) 削除したいブランチを指定する

  $ git branch -d ブランチ名
